# Growth Loop

Work Wiki 不只帮助用户记住过去，还应帮助用户改进。

## Review Levels

1. Daily Capture: 发生了什么，以及哪些证据支持它。
2. Evidence Review: 什么已验证、部分完成、正在探索或仍不确定。
3. Pattern Mining: 哪些模式在周和月之间反复出现。
4. Growth Coaching: 哪些实验或训练动作应该改变行为。
5. Career Narrative: 哪些长期证据可以支持 performance reviews、promotion packets、portfolios 和 resume material。

Version 0 支持第 1 到第 4 层。

## Scorecard

追踪趋势，而不是做惩罚性评分：

- Deep work ratio
- Shipping rate
- Verification quality
- Learning density
- Context switching
- Collaboration load
- Energy signal

## Experiments

每周最多应有一到两个 active experiments。

每个 experiment 应包含：

- State: proposed / active / evaluating / adopted / dropped / promoted
- Hypothesis
- Duration
- Evidence to watch
- Expected behavior change
- Result
- Decision: adopt, revise, or drop

生命周期：

- `proposed` -> `active` or `dropped`
- `active` -> `evaluating` or `dropped`
- `evaluating` -> `adopted`, `dropped`, or `active`
- `adopted` -> `promoted`
- `promoted` 表示该实践已被写入某个长期索引，例如 `indexes/workflows.md`。

## Skill Graph

`indexes/skills.md` 通过 EvidenceRefs、相关项目、实践、当前水平、差距和下一步训练动作来追踪技能。

## Recurring Problems

`indexes/recurring-problems.md` 将反复出现的低效问题作为改进目标来追踪。

## Quality of Life

`growth/quality-of-life.md` 捕捉工作相关信号，例如过载、会议压力、反复阻塞和恢复时间不足。它应保持基于证据，默认不应变成私人日记。
