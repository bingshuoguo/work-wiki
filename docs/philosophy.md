# Philosophy

Work Wiki 存在的原因是：现代工作分布在工具、agent、对话、仓库和文档之间。

与 AI agent 一起工作的人会产生一种新的工作痕迹：prompts、tool calls、code edits、review loops、verification commands 和 repeated context loading。只有当这些痕迹被基于证据地总结并持续维护时，它们才能告诉我们工作实际是如何发生的。

Work Wiki 建立在四个信念之上：

1. 更好的复盘来自证据，而不是记忆。
2. 每日总结只有在能积累成周度和月度模式时才真正有用。
3. Agent-assisted workers 需要改进协作工作流，而不只是提高产出数量。
4. 系统必须提升用户的工作质量，而不能变成监控。

目标是一份持久的 work wiki：本地、私有、由 agent 维护，记录发生了什么、什么重要、什么反复出现，以及下一步应该改变什么。
