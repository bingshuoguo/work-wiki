# Philosophy

Work Wiki exists because modern work is distributed across tools, agents, conversations, repositories, and documents.

People who work with AI agents produce a new kind of work trace: prompts, tool calls, code edits, review loops, verification commands, and repeated context loading. Those traces can teach us how work actually happened, but only if they are summarized with evidence and maintained over time.

Work Wiki is built on four beliefs:

1. Better reviews come from evidence, not memory.
2. Daily summaries are useful only when they accumulate into weekly and monthly patterns.
3. Agent-assisted workers need to improve their collaboration workflows, not just their output volume.
4. The system must improve the user's work quality without becoming surveillance.

The goal is a durable work wiki: a local, private, agent-maintained record of what happened, what mattered, what repeated, and what should change next.
