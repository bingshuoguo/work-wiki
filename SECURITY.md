# Security Policy

Work Wiki 本地优先，并默认关注隐私。不要提交真实证据源注册表、原始证据导出、token、cookie、API key、OAuth secret、私密聊天记录、邮件正文、私有文档、客户数据或专有代码片段。

请通过仓库 issue tracker 报告安全或隐私问题，除非报告中包含敏感信息。如果涉及敏感信息，请在可用时使用 GitHub private vulnerability reporting；否则只创建一个最小 issue，请求私密联系路径，不要包含敏感细节。

发布前，请运行 [docs/validation-plan.md](docs/validation-plan.md) 中的验证检查。
