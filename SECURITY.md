# Security Policy

Work Wiki is local-first and privacy-aware. Do not commit real evidence source registries, raw evidence exports, tokens, cookies, API keys, OAuth secrets, private chat logs, email bodies, private documents, customer data, or proprietary code excerpts.

Report security or privacy issues through the repository issue tracker unless the report contains sensitive information. If sensitive information is involved, use GitHub private vulnerability reporting when available; otherwise open a minimal issue asking for a private contact path without including sensitive details.

Before release, run the validation checks in [docs/validation-plan.md](docs/validation-plan.md).
